# 数据结构与算法的基本概念扫盲

## 所有数据结构和算法都是用来解决实际问题的,所以一定不要脱离实际

## 那么如何衡量数据结构与算法的表现,就是说,这个算法到底好不好？

- 时间，表征程序**执行时长** -> 时间复杂度 -> 只关注某函数执行了多少次!
- 空间，表征程序执行**需要开辟的内存** -> 空间复杂度

以上两个概念,是平衡的艺术,时间换取空间,空间换取时间

### 具体怎么衡量?

```ts
for (let i = 0; i < 100; i++) {
  console.log(i); // 里面的函数执行,跟 for 循环次数有关
}
```

我执行一次是 1ms,那100次就是 1ms ✖️ 10000 ?
并不是! start2: 0.02099609375 ms, start3: 0.016845703125 ms
从这里可以看出,每次执行程序的时间根本就不好衡量,因为计算机配置也不同嘛

为了表征时间的消耗，通常用一次程序执行的单位时间乘以次数来表示时间复杂度，这种方式我们称为**大O计数法 O(n)**

```ts
for (let i = 0; i < 100; i++) {
  console.log(i); // 里面的函数执行,跟 for 循环次数有关
}
```

### 🌟常见时间复杂度

- 常数时间复杂度: O(1)
- 线性时间复杂度: O(n)
- 对数时间复杂度: O(log n)
- O(nlogn)

---

- 平方时间复杂度: O(n²)

### O(1) -> 执行 1 次

```ts
function constantTimeExample(arr: number[]): void {
  console.log(arr[0]);
}

constantTimeExample([1, 2]);
```

### O(n) -> 执行 n 次

```ts
function linearTimeExample(arr: number[]): void {
  for (let i = 0; i < arr.length; i++) {
    console.log(arr[i]);
  }
}

linearTimeExample([1, 2]);
```

### O(log n) -> 执行 log n 次

```ts
function logarithmicTimeExample(n: number): void {
  while (n > 1) {
    console.log(n);
    n = Math.floor(n / 2); // 每次将 n 减少一半
  }
}

logarithmicTimeExample(16);
```

O(logn)

**为什么是 O(log n)？**

关键在于「每次将 n 减半」。假设 n = 16：

- 第 1 次：16 → 8
- 第 2 次：8 → 4
- 第 3 次：4 → 2
- 第 4 次：2 → 1（结束）

循环只执行了 4 次，而 4 = log₂(16)。也就是说：**要执行多少次「除以 2」才能把 n 变成 1？答案是 log₂(n) 次**。所以循环次数 ≈ log n，时间复杂度就是 O(log n)。

对比：n = 1000 时，O(n) 要执行 1000 次，O(log n) 只需约 10 次（2¹⁰ ≈ 1024）。

### O(n log n) -> 执行 n × log n 次

```ts
function nLogNExample(n: number): void {
  for (let i = 0; i < n; i++) {
    let x = n;
    while (x > 1) {
      x = Math.floor(x / 2); // 内层 log n 次
    }
  }
}

nLogNExample(16);
```

O(nlogn)

以上都是正常的

---

### O(n²) -> ☠️到平方就没必要谈论了,性能拉中之拉(若代码中出现双层 for 循环基本可判断是垃圾实现,一定能至少转为O(nlogn))

```ts
function quadraticTimeExample(arr: number[][]): void {
  for (let i = 0; i < arr.length; i++) {
    for (let j = 0; j < arr.length; j++) {
      console.log(arr[i][j]);
    }
  }
}

quadraticTimeExample([1, 2]);
```

## 常见空间复杂度

空间复杂度用于衡量算法在运行过程中所需的内存空间。常见的空间复杂度有：

- S(1)－常数空间
- S(n)－线性空间
- S(n²)－平方空间
